## Objective
Create a NestJS backend service template with production-ready defaults: structured logging (pino), health check endpoint, graceful shutdown, global error handling, environment config, and vitest test setup.

## Scope
### Owns
- `templates/backend-service/nestjs/` (entire directory)
- `templates/backend-service/template.json`

### Reads
- `common/typescript-config/base.json`

## Contracts
### Consumes
- **template-manifest** format from [[TASK-69dd543e-12b8]]

### Publishes
- **nestjs-service-template**: Template files at `templates/backend-service/nestjs/`

## Acceptance Criteria
### Automated
- [ ] Given the template, when `template.json` is inspected, then it defines type `backend-service`, variant `nestjs`, variables `name`, `description`, `port`
- [ ] Given substituted files, when `pnpm install && pnpm build` runs, then it succeeds
- [ ] Given the service, when started, then GET `/health` returns `{ status: 'ok' }` with 200
- [ ] Given the service, when `pnpm test` runs, then vitest executes health endpoint test
- [ ] Given SIGTERM, when sent to the process, then it shuts down gracefully

### Judgment
- [ ] Module structure follows NestJS conventions (module/controller/service)
- [ ] Logging uses nestjs-pino with structured JSON output
- [ ] Error handling returns consistent JSON error responses

## Constraints
- CJS module format (NestJS does not support stable ESM until v12)
- Vitest requires `unplugin-swc` + `.swcrc` for decorator metadata support
- Variables: `{{name}}`, `{{description}}`, `{{port}}` (default 3000)

## Context
NestJS is the default backend framework. Template includes: main.ts (bootstrap with graceful shutdown), app.module.ts (imports ConfigModule, TerminusModule, LoggerModule), app.controller.ts, app.service.ts, health.controller.ts. Uses `@nestjs/config` for env, `@nestjs/terminus` for health, `nestjs-pino` for logging. Testing uses vitest + unplugin-swc due to decorator metadata requirements.
