## Objective
Create a template for new shared platform packages in `common/`. Follows the existing `common/hello-ts` pattern: ESM TypeScript with `exports` field, vitest tests, extending the shared tsconfig base.

## Scope
### Owns
- `templates/platform-package/` (entire directory)

### Reads
- `common/hello-ts/` (reference pattern)
- `common/typescript-config/base.json`

## Contracts
### Consumes
- **template-manifest** format from [[TASK-69dd543e-12b8]] (uses `template.json` schema)

### Publishes
- **platform-package-template**: Template files at `templates/platform-package/`

## Acceptance Criteria
### Automated
- [ ] Given the template, when inspected, then `template.json` defines type `platform-package` with variables `name` and `description`
- [ ] Given the template files, when `{{name}}` and `{{description}}` are substituted, then the result is a valid pnpm package
- [ ] Given the generated package, when `pnpm install && pnpm build` runs, then it succeeds
- [ ] Given the generated package, when `pnpm test` runs, then vitest executes the example test

### Judgment
- [ ] Package structure matches existing `common/` conventions
- [ ] `exports` field in package.json correctly exposes ESM entry point

## Constraints
- Must use `"type": "module"` and `exports` field
- Must extend `@common/typescript-config/base.json`
- Test file at `__tests__/unit/index.test.ts`

## Context
Follows the existing `common/hello-ts` package as a reference. Key additions over that pattern: `exports` field, vitest config, and `template.json` manifest for the scaffold tool.
