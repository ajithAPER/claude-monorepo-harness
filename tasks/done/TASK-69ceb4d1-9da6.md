# Migrate package manager from Yarn 4 to PNPM

## Objective

Replace Yarn 4 (Berry) with PNPM as the monorepo package manager. All workspace orchestration, dependency resolution, and developer-facing commands must work identically after migration.

## Scope

### Owns
- `package.json` (root) — update `packageManager`, `scripts`, remove `@yarnpkg/types` devDep
- `yarn.config.cjs` — DELETE (Yarn constraints file)
- `yarn.lock` — DELETE
- `.yarnrc.yml` — DELETE
- `.yarn/` — DELETE directory
- `.gitignore` — update Yarn Berry section to PNPM equivalent
- `.gitattributes` — remove Yarn-specific entries
- `pnpm-workspace.yaml` — CREATE (workspace config)
- `pnpm-lock.yaml` — CREATE (generated by `pnpm install`)
- `.npmrc` — CREATE if needed (equivalent of .yarnrc.yml settings)
- `README.md` — update prerequisites, install command, and directory tree references
- `tools/code-memory/README.md` — update `yarn workspace` command references
- `tools/code-memory/src/config.mjs` — update `.yarn` ignore pattern to PNPM equivalent
- `.claude/settings.json` — update `Bash(yarn:*)` permission to `Bash(pnpm:*)`
- `.claude/settings.local.json` — update all `Bash(yarn ...)` permissions to `Bash(pnpm ...)`

### Reads
- `tools/code-memory/package.json` — verify workspace name for pnpm commands
- All `products/*/apps/*/package.json` — verify workspace glob compatibility
- `context/decisions.md` — for recording the migration decision

## Contracts

### Consumes
None

### Publishes
None (internal tooling change, no API contract changes)

## Acceptance Criteria

### Automated
- [x] `pnpm install` succeeds from a clean state (no `node_modules/`)
- [x] `pnpm run build` executes workspace builds (same as `yarn workspaces foreach -Apt run build`)
- [x] `pnpm run test` executes workspace tests
- [ ] `pnpm run check` executes workspace checks
- [x] No files named `yarn.lock`, `.yarnrc.yml`, or `yarn.config.cjs` exist in the repo
- [x] No `.yarn/` directory exists (except possibly `.yarn/releases` can be removed)
- [x] `pnpm-lock.yaml` is committed
- [x] `pnpm-workspace.yaml` defines the same workspace globs as the old `package.json.workspaces`
- [x] `package.json` has `packageManager` set to a PNPM version
- [x] `grep -ri yarn README.md` returns no results (all references updated)
- [x] `grep -ri yarn tools/code-memory/README.md` returns no results

### Judgment
- [x] Migration decision recorded in `context/decisions.md`
- [x] `.gitignore` entries are appropriate for PNPM (no leftover Yarn patterns)
- [x] `.claude/settings.json` and `.claude/settings.local.json` permissions reference `pnpm` not `yarn`

## Constraints

- Must preserve the `node-modules` linker strategy (no PnP or symlink hoisting that would break native addons like `better-sqlite3` and `tree-sitter`)
- Must preserve workspace topology: `tools/*`, `common/*`, `products/*/apps/*`
- Do NOT change any workspace-level `package.json` files (only root)
- PNPM version should be latest stable (use `corepack enable` + `corepack prepare`)

### Migration steps (recommended order)
1. Delete Yarn artifacts: `.yarnrc.yml`, `yarn.config.cjs`, `yarn.lock`, `.yarn/`
2. Create `pnpm-workspace.yaml` and `.npmrc`
3. Update root `package.json` (`packageManager`, scripts, devDeps)
4. Run `pnpm install` to generate lockfile
5. Update documentation (README.md, tools/code-memory/README.md)
6. Update `.gitignore` and `.gitattributes`
7. Update `.claude/settings.json` and `.claude/settings.local.json`
8. Update `tools/code-memory/src/config.mjs` if needed
9. Verify: clean install, build, test
10. Record decision in `context/decisions.md`
