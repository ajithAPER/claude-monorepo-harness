## Objective
Create `tools/scaffold` — a pnpm package with MJS+JSDoc bin scripts that scaffolds new apps, services, and packages from templates. Handles template discovery, variant selection, variable prompting, file copying with substitution, workspace registration, and `pnpm install`.

## Scope
### Owns
- `tools/scaffold/` (entire directory)
- `templates/` (directory creation, template manifest format)

### Reads
- `pnpm-workspace.yaml`
- `apps/pnpm-workspace.yaml`
- `common/*/package.json`

## Contracts
### Consumes
- **workspace-layout** from [[TASK-69dd543e-a3f2]]

### Publishes
- **template-manifest**: Each template has a `template.json` defining `type`, `variants`, `variables`, and `defaultVariant`
- **scaffold-cli**: `pnpm scaffold <type> [name]` — interactive prompts for variant + variables

## Acceptance Criteria
### Automated
- [ ] Given `tools/scaffold/package.json`, when inspected, then it defines a `bin` entry and `"type": "module"`
- [ ] Given a template directory with `template.json` and file tree, when `scaffold` runs with answers, then files are copied to target with all `{{variable}}` placeholders replaced
- [ ] Given scaffolding a `platform-package`, when complete, then the package is registered in root `pnpm-workspace.yaml` under `common/*`
- [ ] Given scaffolding an `app`, when complete, then it is registered in `apps/` nested workspace
- [ ] Given scaffolding completes, when `pnpm install` runs, then it succeeds
- [ ] Given a template with variants, when the user selects a variant, then only that variant's files are copied

### Judgment
- [ ] CLI prompts are clear and intuitive
- [ ] Error messages are helpful (missing template, invalid name, etc.)
- [ ] Code follows MJS+JSDoc conventions (no TypeScript in tooling)

## Constraints
- MJS with JSDoc types only — no TypeScript
- No heavy dependencies — prefer `node:readline` or lightweight prompt lib
- Must support `--name`, `--variant` flags for non-interactive use
- Template variable format: `{{variableName}}` in file contents and filenames

## Context
The scaffold tool is the main user-facing entry point for creating new workspace packages. It reads template definitions from `templates/`, prompts for missing values, copies files with substitution, updates workspace config, and runs `pnpm install`. Template types are discovered dynamically from the `templates/` directory.
